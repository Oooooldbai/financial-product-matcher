## Rules层核心内容

### 一、匹配规则框架（RULES.md）

**两大核心场景**：
1. **以产品为中心**：了解产品 → 筛选客户 → 分析切入点 → 评估风险 → 制定话术
2. **以客户为中心**：了解客户 → 匹配产品 → 评估适配 → 评估风险 → 制定话术

**五大规则模块**：
1. 客户画像标准（必填/选填字段）
2. 产品匹配矩阵（企业性质×行业×规模×供应链位置）
3. 适配点分析规则（现金流/账期/收益/融资成本/供应商关系）
4. 话术路由规则（客户类型×企业性质×产品×切入角度）
5. 风险判断规则（准入门槛/合规检查）

---

### 二、匹配引擎（MATCHING_ENGINE.md）

**引擎架构**：
```
输入 → 预处理 → 匹配计算 → 结果输出
```

**两大算法**：
1. **以客户为中心的产品推荐**
   - 筛选候选产品
   - 多维度打分（痛点/需求/准入条件/利率/核心企业关联/行业经验）
   - 输出推荐列表

2. **以产品为中心的客户筛选**
   - 提取产品特征
   - 筛选候选客户
   - 输出客户列表

**匹配度计算公式**：
```
总分 = 痛点匹配度×0.30
     + 需求匹配度×0.25
     + 准入条件匹配×0.20
     + 利率优势×0.10
     + 核心企业关联×0.10
     + 行业经验×0.05
```

**话术库结构**：
```
四维度路由：
1. 客户类型：资产端（核心企业）/ 资金端（银行/保理）
2. 企业性质：民营/国央企/大型民企
3. 产品类型：DD/SCF/保理/订单贷/其他
4. 切入角度：痛点/利益/案例/合规/关系/资产质量/风险控制/时效/成本
```

## Rules层使用流程

### 流程图

```
Step 1: 输入
  ├─ 客户画像 OR
  └─ 产品ID

Step 2: 匹配
  ├─ 筛选候选产品/客户
  ├─ 多维度打分
  └─ 生成推荐列表

Step 3: 分析
  ├─ 适配点分析
  ├─ 风险评估
  └─ 话术选择

Step 4: 输出
  ├─ 推荐产品/客户列表
  ├─ 匹配理由
  ├─ 切入角度
  ├─ 推荐话术
  └─ 风险提示
```

### 使用示例

**输入：客户画像**
```json
{
  "customer_name": "深圳XX电子科技",
  "enterprise_type": "民企",
  "industry": "电子制造",
  "scale": {"annual_revenue": "5000万"},
  "supply_chain_position": "上游供应商",
  "core_enterprise": "华为",
  "pain_points": ["应收账款周期长", "订单增长但资金不足", "无抵押物"],
  "needs": ["快速回笼资金", "无抵押融资", "灵活额度"]
}
```

**匹配引擎处理**：
1. 筛选候选产品：华为供应商适配的产品
2. 多维度打分：痛点30分+需求25分+准入20分+利率10分+核心企业10分+行业5分=92分
3. 话术选择：客户类型（资产端）+企业性质（民营）+产品（DD）+切入角度（痛点）= 话术A1
4. 输出推荐列表

**输出：推荐结果**
```json
{
  "customer_id": "CUST-001",
  "recommendations": [
    {
      "product_id": "PROD-001",
      "product_name": "信e链",
      "bank": "中信银行",
      "score": 92,
      "match_reasons": [
        "完美匹配上游供应商身份",
        "解决应收账款周期长的痛点",
        "无需抵押物，符合客户需求",
        "华为核心企业有授信额度"
      ],
      "entry_angle": "痛点切入",
      "script": "话术A1：民营上市企业×DD×痛点切入",
      "risk_alert": "需确认贸易背景真实性"
    }
  ]
}
```
