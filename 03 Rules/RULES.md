# Rules层 - 匹配规则框架

> 金融产品智能匹配工具的核心算法层

---

## 核心定位

Rules层是匹配工具的"大脑"，负责：
1. 接收输入（客户或产品）
2. 匹配知识库
3. 输出推荐结果

---

## 两大核心场景

### 场景一：以产品为中心
```
了解产品 → 筛选客户 → 分析切入点 → 评估风险 → 制定话术
```

**适用**：拿到新产品，想快速找到目标客户

### 场景二：以客户为中心
```
了解客户 → 收集数据 → 匹配产品 → 评估适配 → 制定话术
```

**适用**：接触新客户，想知道推什么产品

---

## 场景一：以产品为中心的匹配流程

### Step 1: 了解产品
```
输入: 产品名称/产品类型
↓
输出: 产品画像
- 产品定义
- 适用客户类型
- 准入条件
- 利率区间
- 期限范围
- 担保要求
- 风控要点
```

### Step 2: 筛选客户
```
输入: 产品画像
↓
匹配规则:
- 企业性质筛选
- 行业筛选
- 规模筛选
- 供应链位置筛选
↓
输出: 候选客户列表（Top N）
```

### Step 3: 分析切入点
```
输入: 产品画像 + 客户画像
↓
分析:
- 产品为什么适合这个客户
- 客户目前的痛点是什么
- 产品能解决客户的什么问题
↓
输出: 适配点分析
- 核心适配点
- 次要适配点
```

### Step 4: 评估风险
```
输入: 产品画像 + 客户画像 + 业务场景
↓
合规检查:
- 牌照要求
- 利率合规
- 贸易背景真实性
- 财务处理
- 税务处理
↓
输出: 风险评估报告
- 合规风险点
- 财务风险点
- 操作风险点
```

### Step 5: 制定话术
```
输入: 产品画像 + 客户画像 + 切入点
↓
话术生成:
- 选择话术模板
- 填入客户具体数据
- 生成个性化话术
↓
输出: 营销话术脚本
```

## 场景二：以客户为中心的匹配流程

### Step 1: 了解客户
```
输入: 客户名称/客户类型
↓
数据收集提示:
基础信息:
- 企业名称、行业、成立时间
- 企业性质（民营/国央企/外资）
- 规模（年营收/年应付账款）

财务数据:
- 年营收、年应付账款
- 平均账期
- 资产负债率
- 现金流状况

业务特征:
- ERP系统
- 供应商数量
- 核心企业关系

信用状况:
- 信用评级
- 历史违约记录
- 银行授信额度

↓
输出: 客户画像
```

### Step 2: 匹配产品
```
输入: 客户画像
↓
匹配规则:
- 企业性质×产品适配
- 行业×产品适配
- 规模×产品适配
- 供应链位置×产品适配
- 财务指标×产品适配
↓
输出: 推荐产品列表（Top N）
- 产品名称
- 匹配度
- 推荐理由
```

### Step 3: 评估适配
```
输入: 客户画像 + 推荐产品
↓
适配分析:
- 产品为什么适合这个客户
- 能解决客户的什么问题
- 客户的现状 vs 产品要求
↓
输出: 适配点分析
- 核心适配点
- 次要适配点
- 差距分析
```

### Step 4: 评估风险
```
输入: 客户画像 + 产品 + 业务场景
↓
风险检查:
- 合规风险（牌照/利率/贸易背景）
- 财务风险（会计处理/报表影响）
- 税务风险（增值税/所得税）
- 政策风险（行业监管/特殊规定）
↓
输出: 风险评估报告
- 合规风险点
- 财务风险点
- 税务风险点
- 应对建议
```

### Step 5: 制定话术
```
输入: 客户画像 + 产品 + 切入点
↓
话术生成:
- 选择话术模板（客户类型×产品类型×切入角度）
- 填入客户具体数据
- 生成个性化话术
↓
输出: 营销话术脚本
- 开场话术
- 痛点话术
- 方案话术
- 异议处理
```

## 匹配规则核心逻辑

### 匹配输入
```
客户画像 = {
  企业性质: 民营/国央企/外资,
  行业: 制造业/零售/医药/科技/汽车/其他,
  规模: 大型/中型/小型/微型,
  供应链位置: 核心企业/一级供应商/二级供应商/非核心,
  财务指标: {
    年营收: xxx,
    年应付账款: xxx,
    平均账期: xxx天,
    资产负债率: xx%,
    现金流: 充裕/紧张/正常
  },
  信用状况: {
    评级: AAA/AA/A/BBB/其他,
    融资成本: xx%,
    银行授信: xxx
  },
  融资痛点: 现金流/账期/供应商关系/融资成本,
  当前合作方: xxx
}
```

### 匹配输出
```
推荐结果 = {
  匹配度: xx%,
  推荐产品: [产品列表],
  适配点: [为什么匹配],
  切入角度: [从哪里切入],
  推荐话术: [具体话术],
  风险提示: [合规/风险要点]
}
```

---

## 一、客户画像标准

### 必填字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 企业性质 | 枚举 | 民营上市/民营非上市/国央企/外资 |
| 行业 | 枚举 | 制造业/零售/医药/科技/汽车/建筑/物流/其他 |
| 规模 | 枚举 | 大型/中型/小型/微型 |
| 供应链位置 | 枚举 | 核心企业/一级供应商/二级供应商/其他 |
| 年应付账款 | 数值 | 单位：亿元 |

### 选填字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 平均账期 | 数值 | 单位：天 |
| ERP系统 | 枚举 | SAP/Oracle/用友/金蝶/其他 |
| 信用评级 | 枚举 | AAA/AA/A/BBB/无评级 |
| 融资成本 | 数值 | 年化利率% |
| 主要痛点 | 多选 | 现金流/账期/供应商/融资成本 |

---

## 二、产品匹配矩阵

### 维度权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 企业性质 | 25% | 不同性质适配不同产品 |
| 行业 | 20% | 行业特征影响产品选择 |
| 规模 | 20% | 大小客户适配不同产品 |
| 供应链位置 | 20% | 核心企业和供应商适配不同 |
| 财务指标 | 15% | 具体指标影响产品匹配 |

### 匹配规则

#### 规则1: 企业性质×产品适配

| 企业性质 | 优先产品 | 次选产品 | 适配说明 |
|----------|----------|----------|----------|
| 民营上市 | DD/SCF | 保理/ABS | 收益导向，灵活度高 |
| 民营非上市 | 保理/小贷 | SCF | 融资需求强，门槛适中 |
| 国央企 | SCF | 保理 | 合规导向，账期管理 |
| 外资 | DD/保理 | SCF | 收益导向，流程规范 |

#### 规则2: 行业×产品适配

| 行业 | 优先产品 | 账期特点 | 说明 |
|------|----------|----------|------|
| 汽车 | SCF/票据 | 60-120天 | 账期长，适合SCF |
| 家电 | DD/SCF | 30-90天 | 灵活切换 |
| 医药 | SCF | 90-180天 | 账期极长 |
| 零售 | DD | 30-60天 | 短账期，DD效率高 |
| 制造业 | SCF/DD | 60-90天 | 视现金流而定 |
| 科技 | 保理/小贷 | 30-90天 | 轻资产，适合保理 |

#### 规则3: 规模×产品适配

| 规模 | 年应付门槛 | 优先产品 | 说明 |
|------|------------|----------|------|
| 大型 | ≥10亿 | DD/SCF/ABS | 多产品组合 |
| 中型 | 1-10亿 | SCF/保理 | 单产品深耕 |
| 小型 | <1亿 | 保理/小贷 | 融资需求强 |
| 微型 | <1000万 | 小贷/信用卡 | 门槛最低 |

#### 规则4: 供应链位置×产品适配

| 位置 | 核心产品 | 资金来源 | 说明 |
|------|----------|----------|------|
| 核心企业 | DD/SCF | 自有/银行 | 主导方 |
| 一级供应商 | SCF/保理 | 银行/保理 | 背靠核心企业 |
| 二级供应商 | 保理 | 保理公司 | 信用弱，担保要求 |
| 非核心 | 小贷/民间 | 民间资金 | 门槛最低 |

## 三、适配点分析规则

### 适配点类型

| 适配点 | 适用场景 | 推荐产品 | 话术角度 |
|--------|----------|----------|----------|
| 现金流优化 | 供应商资金紧张 | SCF/保理 | "缓解供应商资金压力" |
| 账期管理 | 核心企业想延长账期 | DD | "0成本延长账期" |
| 收益提升 | 核心企业现金充裕 | DD | "躺睡资金变收益" |
| 融资成本 | 供应商融资成本高 | SCF | "用核心企业信用融资" |
| 供应商关系 | 想维护供应商关系 | DD/SCF | "帮助供应商就是帮自己" |

### 适配点判断规则

```
IF 核心企业现金充裕 AND 账期≥60天:
  推荐 = DD
  适配点 = ["收益提升", "账期管理"]
  切入角度 = "您账上趴着的应付账款，每年白白躺60-90天"

IF 核心企业现金紧张 AND 供应商分散:
  推荐 = SCF
  适配点 = ["现金流优化", "供应商关系"]
  切入角度 = "帮助供应商获得低成本融资，稳定供应链"

IF 供应商信用弱 AND 融资成本高:
  推荐 = 保理
  适配点 = ["融资成本"]
  切入角度 = "用应收账款获得快速融资"
```

## 四、话术路由规则

### 路由维度

| 维度 | 选项 | 说明 |
|------|------|------|
| 客户类型 | 资产端/资金端 | 核心企业=资产端，银行/保理=资金端 |
| 企业性质 | 民营/国央企/大型民企 | 影响话术风格 |
| 推荐产品 | DD/SCF/保理/其他 | 影响话术内容 |
| 切入角度 | 收益/合规/关系/成本 | 影响话术重点 |

### 话术选择矩阵

| 客户类型 | 企业性质 | 产品 | 推荐话术类型 |
|----------|----------|------|--------------|
| 资产端 | 民营上市 | DD | 收益导向话术 |
| 资产端 | 国央企 | SCF | 合规导向话术 |
| 资产端 | 大型民企 | DD/SCF | 收益+关系话术 |
| 资金端 | 银行 | SCF资产 | 资产质量话术 |
| 资金端 | 保理 | 保理资产 | 风险控制话术 |

## 五、风险判断规则

### 准入门槛

| 产品 | 最低门槛 | 禁止条件 |
|------|----------|----------|
| DD | 年应付≥1亿，核心企业评级≥AA | 高污染/强周期行业 |
| SCF | 年应付≥5000万，核心企业评级≥A | 虚构贸易背景 |
| 保理 | 企业成立≥2年，有稳定下游 | 下游集中度>80% |
| ABS | 年应收≥5亿，评级≥AA | 非适格基础资产 |

### 合规检查

| 检查项 | 规则 | 风险等级 |
|--------|------|----------|
| 牌照要求 | 经营主体是否有对应牌照 | 高 |
| 利率合规 | LPR×4上限 | 高 |
| 贸易背景 | 是否有真实贸易合同+发票 | 高 |
| 国企采购 | 是否符合国资委采购规定 | 中 |
| 数据合规 | 是否符合个人信息保护法 | 中 |

## 六、优先级排序规则

### 匹配度计算

```
匹配度 = 
  企业性质匹配度 × 25% +
  行业匹配度 × 20% +
  规模匹配度 × 20% +
  供应链位置匹配度 × 20% +
  财务指标匹配度 × 15%
```

### 推荐排序

```
1. 匹配度≥80%: 强烈推荐
2. 匹配度60-80%: 推荐
3. 匹配度40-60%: 可选
4. 匹配度<40%: 不推荐
```

## 工具使用示例

### 输入
```
客户画像:
  企业性质: 民营上市
  行业: 汽车
  规模: 大型
  供应链位置: 核心企业
  年应付账款: 200亿
  平均账期: 90天
  ERP: SAP
  信用评级: AAA
  主要痛点: [现金流, 账期]
```

### 输出
```
推荐产品:
1. DD (匹配度92%)
   适配点: 账期管理(90天)、收益提升(躺睡资金变收益)
   切入角度: "您账上趴着的应付账款，每年白白躺90天。现在可以把它变成年化15-25%的无风险投资收益。"
   话术: [民营上市企业DD话术模板]
   风险: 无重大风险
   
2. SCF (匹配度85%)
   适配点: 供应商关系、现金流优化
   切入角度: "帮助供应商获得低成本融资，稳定供应链"
   话术: [民营上市企业SCF话术模板]
   风险: 需要确认核心企业信用评级
```
