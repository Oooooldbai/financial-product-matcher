# 话术库 - 核心话术模板

> 金融产品智能匹配工具 - 话术路由与生成

---

## 话术库结构

话术库按**四维度路由**：
1. **客户类型**：资产端（核心企业）/ 资金端（银行/保理）
2. **企业性质**：民营/国央企/大型民企
3. **产品类型**：DD（数字债务凭证）/ SCF（供应链金融）/ 保理/ 订单贷/ 其他
4. **切入角度**：痛点切入/利益切入/案例切入/时效切入

## 一、资产端话术（核心企业）

### 类型A：民营上市企业 × DD × 收益导向

#### 🔥 痛点切入：资金利用效率低

**适用场景**：
- 客户账期60-90天
- 年应付账款规模大（>10亿）
- 现金流充裕
- 痛点：应付账款趴账，资金闲置

**开场话术**：
```
王总，我跟您聊聊一个可能被您忽视的赚钱机会。

您现在应付账款一般多少天？
（客户回答90天）
那这90天里，这笔钱在您账上就是"躺睡"状态，对吧？
（客户点头）

您年应付账款规模大概多少？
（客户回答50亿）
如果按50亿算，这50亿趴账90天，相当于每年有150亿的现金流闲置。

现在有个工具，能让这150亿从"躺睡"变成"躺赚"，您感兴趣吗？
```

**痛点深挖**：
```
痛点一：资金利用效率低
- 应付账款趴账60-180天
- 沉淀资金规模大
- 资金成本高（如发债成本4-6%）
- 躺睡资金=隐形亏损

痛点二：财务报表压力
- 应付账款规模大影响负债率
- 现金流管理压力大
- 银行授信占用

痛点三：机会成本
- 现金充裕但找不到好投资
- 低风险收益渠道有限
- 资金闲置=机会成本
```

**方案话术**：
```
我们有个产品叫"数字债务凭证"（DD）。

简单说，就是帮您把趴账的应付账款，转换成一个可以产生收益的凭证。

核心逻辑：
1. 您原本需要90天付款给供应商
2. 现在我们给供应商提供即时付款（用银行资金）
3. 您给我们开一个电子凭证，约定90天后付款
4. 期间，这个凭证可以在市场上流转，产生收益

对您的好处：
✅ 0成本延长账期（账上资金不变）
✅ 躺睡资金变收益（年化10-25%）
✅ 不占用银行授信
✅ 稳定供应链关系（供应商有钱收）

比如您年应付50亿，用DD：
- 50亿×90天=125亿资金沉淀
- 按15%年化收益=18.75亿/年
- 这就是您以前白白损失的钱
```

**异议处理**：

**异议1：这个凭证风险高吗？**
```
王总，您担心风险很正常。

首先，这个凭证是"数字债务凭证"，有：
- 区块链存证，不可篡改
- 银行或核心企业信用背书
- 监管认可，金融基础设施支持

其次，风险可控：
- 凭证到期自动兑付
- 不存在违约风险（核心企业AAA信用）
- 监管全程监控

最后，我们平台：
- 严格准入（只接优质核心企业）
- 流程透明（每笔可查）
- 保险保障（部分产品有保险）
```

**异议2：供应商不愿意怎么办？**
```
王总，这是普遍担心，但实际上：

1. 供应商愿意收DD凭证：
   - 能快速回笼资金（T+0到账）
   - 利率比银行低（3-6%）
   - 不用等您账期（您90天，供应商T+0）

2. 供应商选择：
   - 持有到期：等您90天付款
   - 贴现流转：现在就拿到钱
   - 分级持有：部分贴现部分持有

3. 例子：
   某汽车零部件供应商：
   - 原本等90天才能回款
   - 现在用DD凭证T+0到账
   - 成本从10%降到5%
   - 资金周转加快3倍
```

**异议3：这个操作流程复杂吗？**
```
王总，流程非常简单：

第一步：系统对接（一次性）
   - 我们和您的ERP系统对接
   - 自动读取应付账款数据
   - 无需人工操作

第二步：发起付款
   - 您需要付款给供应商时，在系统里选"DD凭证支付"
   - 系统自动生成电子凭证
   - 供应商T+0收到资金

第三步：到期兑付
   - 90天后，系统自动从您账户扣款
   - 完成兑付
   - 全流程线上，无需见面

整个流程：
- 首次对接：2-3天
- 日常使用：1分钟发起
- 到期兑付：自动扣款
```

**促成话术**：
```
王总，总结一下：

1. 您现在每年有150亿资金躺睡90天
2. 用DD，这150亿可以产生18.75亿收益
3. 操作简单，一次性对接后自动运行
4. 风险可控，核心企业信用背书

要不要我们先做个测算？
只需要您提供：
- 年应付账款规模
- 平均账期
- 银行授信利率

我帮您算一下具体能省多少钱。
```
