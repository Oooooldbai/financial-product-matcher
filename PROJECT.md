# 金融产品智能匹配工具项目

> 业务人员随身产品助手 | To B工具 | 2026-04-11启动

---

## 项目概述

### 目标
做一个面向全市场的**金融产品智能匹配工具**（To B工具）

### 核心价值
帮助金融业务人员（尤其是新手）：
- 输入客户画像 → 推荐适配产品
- 知道怎么切入、用什么话术
- 了解合规风险和政策要点

### 定位
不是给某公司用的内部工具，而是面向全市场金融业务人员的SaaS工具

---

## 核心架构

```
客户库（独立） × 产品库（独立） = 多对多匹配
      ↓
   匹配规则层（核心算法）
      ↓
适配点分析 + 切入角度 + 话术生成 + 风险评估
```

### 五大独立库

| 库 | 作用 | 状态 |
|----|------|------|
| **客户库** | 客户画像标准、分类体系、标签系统 | 📋 待完善 |
| **产品库** | 产品手册、准入条件、利率期限 | 📋 待完善 |
| **匹配规则层** | 客户↔产品多对多匹配逻辑 | 🚧 待建设 |
| **话术库** | 按客户类型×产品类型×切入角度路由 | 📋 待完善 |
| **政策法规库** | 每个产品关联对应法规、财务、审计 | 📋 待完善 |

## 工具核心算法

```
输入1：客户画像 → 匹配规则层 → 产品库检索 → 输出：推荐产品列表（Top N）
输入2：产品类型 → 匹配规则层 → 客户库检索 → 输出：推荐客户列表（Top N）
输入3：客户+产品 → 适配点分析库 + 切入角度库 → 输出：适配点+切入角度
输入4：客户+产品+切入角度 → 话术库 → 输出：个性化话术
输入5：产品+客户+业务场景 → 产品-法规关联库 + 风险判断库 → 输出：风险评估报告+合规建议
```

## 🔧 供应链关系发现器（客户库数据采集模块）

> **定位**：金融产品匹配工具的**数据采集分支**，用于解决"不知道一家公司是谁的上下游"这个问题

### 背景问题
- 上下游是**相对关系**，非公司固有属性
- 同一家公司可能是A的上游、B的下游
- 业务人员往往不知道客户的核心企业关联

### 核心功能
自动从公开数据源发现企业的供应链关系：
1. **上市公司年报** → 前五大客户/供应商（强制披露）
2. **招标公告** → 中标供应商关系
3. **工商数据** → 股东穿透、子公司关系
4. **B2B平台** → 采购/供应关系

### 数据输出格式
```json
{
  "company_name": "深圳XX电子",
  "supply_chain_relations": [
    {
      "core_enterprise": "华为",
      "position": "上游供应商",
      "evidence_source": "年报",
      "evidence_url": "...",
      "confidence": "high"
    }
  ]
}
```

### 与主项目的关系
```
金融产品匹配工具（主项目）
├── 产品库（已有采集器）
├── 客户库
│   ├── 基础画像（已有模板）
│   └── ⚡ 供应链关系 ← 供应链发现器（新增分支）
├── 匹配引擎 ✅ 已完成
├── 话术库
└── 政策法规库
```

### 优先级
- **P0**：上市公司年报大客户挖掘（最可靠）
- **P1**：招标公告关系发现
- **P2**：工商数据穿透
- **P3**：B2B平台数据
